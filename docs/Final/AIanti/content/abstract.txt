The rapid evolution of modern healthcare systems and the increasing complexity of clinical environments demand a high level of preparedness, clinical reasoning, and quick decision-making from nursing professionals. Traditional pedagogical methods in nursing education, which heavily rely on didactic lecturing, passive reading, and static multiple-choice question sheets, often fail to stimulate active recall and fail to prepare students for high-stakes, real-world clinical environments. This study addresses these educational limitations by designing, developing, and evaluating NurseQuest, a gamified web-based education and assessment platform specifically tailored to the nursing curriculum. Built using a modern full-stack web architecture—comprising a React 19 single-page application frontend, a Node.js Express 5 backend API server, and a stateful Socket.IO real-time communication layer powered by a Supabase PostgreSQL database—NurseQuest bridges the gap between traditional learning and interactive engagement. The platform implements a unified gamification engine featuring experience points (XP), a 7-tier nursing rank progression system, active streaks, and milestone achievements to motivate continuous learning.

Crucially, the platform supports nine specialized, clinically relevant question formats, including audio-based auscultation challenges, image-based ECG waveform recognition, sequence ordering for medical procedures, and slider-based vital sign estimations, moving far beyond standard text-only multiple-choice trivia. Furthermore, to enable zero-setup classroom implementation, the platform incorporates a live multiplayer session mode where instructors can host competitive quiz lobbies joined by students via 6-digit codes. The backend handles synchronization, real-time response curve grading, and instant leaderboard updates with sub-200ms latency. The system also includes an automated, regular-expression-based quiz document parser that ingests DOCX, PDF, and TXT files, extracting mixed question types and resolving media ZIP bundles automatically. Experimental evaluation under synthetic load testing demonstrated that the Socket.IO server maintains stable state synchronization with an average broadcast latency of 14.2 milliseconds for up to 100 concurrent players, outperforming HTTP polling fallbacks by a factor of thirty. Database benchmarking also confirmed that local transactional queries execute with minimal overhead, validating the platform's suitability for deployment in institutional environments. Pedagogical evaluation guidelines and content validity index mappings further prove the platform's utility in reinforcing clinical knowledge, enhancing engagement, and promoting collaborative learning in undergraduate nursing programs.
