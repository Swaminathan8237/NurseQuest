=== SLIDE 9 ===
Title: PROBLEM STATEMENT
Content:
9
Content:
Traditional nursing education relies on passive learning methods (lectures, static PDFs, paper-based tests), leading to low student engagement, limited real-time assessment, and no personalized feedback — especially in clinical skill-building scenarios.
Lack of engagement
No competitive assessment
Limited question formats
Poor accessibility

A full-stack web platform with gamified learning (XP, ranks, badges), real-time multiplayer quizzes via WebSockets, 6 multimedia question types, role-based access (Teacher/Student), and modular curriculum management — built with React 19, Node.js/Express 5, Socket.IO, and SQLite.

To create a Web App or an Website for Quiz platform which includes Nursing specific question modules that any Nursing Student can attend irrespective of their Syllabus. 
We’ve used React+vite and the middleware as Node+Express.js. RESTful API handles the routing, middleware routing, file uploads and DB operations.
Using SQLite (better-sqlite) for serverless, lightweight use for Database.  
Web Sockets to enable live, real-time multiplayerquiz sessions where a teacher hosts and students join via a game code.

=== SLIDE 10 ===
Title: OBJECTIVES
Content:
10
Content:
Objective  - 1 :
To design a full-stack gamified nursing education platform (React, Node.js, Express, SQLite) with role-based access control, interactive dashboards, and a comprehensive quiz management system. 
Objective  - 2:
To implement multimedia-rich question types — MCQ, image-based, video-based, audio-based, jumbled-letter, and sequence ordering — enabling varied and engaging assessments beyond traditional text-only quizzes.
Objective  - 3:
To integrate nursing-specific learning features including categorized clinical quizzes, module-based organization, scenario-based assessments, and domain-relevant gamification (XP, levels, streaks, badges).
Objective  - 4:
To enable real-time multiplayer quiz sessions via Socket.IO with multi-device access, instant scoring, live leaderboards, and synchronized question delivery.
Objective  - 5:
To ensure production-readiness through robust security (JWT, bcrypt, input validation), scalable single-server architecture, and real-world deployment
